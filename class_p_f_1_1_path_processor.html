◆ DestroyNode()

void PF.PathProcessor.DestroyNode ( GraphNode  node)

Destroyes the given node. This is to be called after the node has been disconnected from the graph so that it cannot be reached from any other nodes. It should only be called during graph updates, that is when the pathfinding threads are either not running or paused.

警告
This method should not be called by user code. It is used internally by the system.

◆ PausePathfinding()

GraphUpdateLock PF.PathProcessor.PausePathfinding ( bool  block)

Prevents pathfinding threads from starting to calculate any new paths.

参数
blockIf true, this call will block until all pathfinding threads are paused. otherwise the threads will be paused as soon as they are done with what they are currently doing.
返回
A lock object. You need to call Unlock on that object to allow pathfinding to resume.
注解
In most cases this should not be called from user code.

◆ NumThreads

int PF.PathProcessor.NumThreads
get

Number of parallel pathfinders. Returns the number of concurrent processes which can calculate paths at once. When using multithreading, this will be the number of threads, if not using multithreading it is always 1 (since only 1 coroutine is used).
